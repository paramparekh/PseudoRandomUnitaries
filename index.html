Basic Quantum We'll Need

Quantum States (Qubits): Unlike classical computing, quantum bits can exist in combinations via superposition.

$| \psi \rangle = \alpha|0\rangle + \beta|1\rangle = \begin{pmatrix} \alpha \\ \beta \end{pmatrix}$

Manipulating States:

$\rightarrow$ We apply unitary transformations (geometric rotations)

The Mathematical View:

  • An $n$-qubit state is a $2^n$-dimensional vector.
  • An $n$-qubit operation is a $2^n \times 2^n$ unitary matrix.

Executing a gate simply rotates the vector

The Problem: Truly Random Functions Are Exponential

To specify $f:\{0,1\}^n \to \{0,1\}^n$ we must store its entire truth table — one $n$-bit output per input:

$$\underbrace{n}_{\text{bits/output}} \;\times\; \underbrace{2^n}_{\text{\# inputs}} \;=\; n\cdot 2^n \;\text{bits}$$

Both are exponential in $n$. No PPT algorithm can store, read, or use such an object.

$n$ $2^n$ inputs $n\cdot 2^n$ bits
10 1,024 10,240
50 $\approx 10^{15}$ $\approx 10^{16.7}$
128 $\approx 10^{38}$ $\approx 10^{40}$

atoms in observable universe $\approx 10^{80}$

Infeasible for PPT Algorithms

No probabilistic polynomial-time algorithm can store, read, or use a truly random function as a cryptographic building block.  $\Rightarrow$  We need a computationally indistinguishable substitute.
